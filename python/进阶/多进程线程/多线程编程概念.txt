并发 # 一个cpu极短时间轮流循环处理多个进程

并行 # 多个cpu处理多个进程

程序 # 一个程序至少有一个进程

进程 # 分配资源最小单元,系统会给运行中的进程分配cpu,内存,磁盘,网络等
主进程 # 子进程执行完毕后,主进程才会结束
子进程 # 由主进程创建的进程

线程 # 程序执行的最小单元,一个进程至少有一个线程,没有进程就没有线程
主线程 # 由操作系统创建的线程
主线程 # 子线程执行完毕后,主线程才会结束
子线程 # 由主线程创建的线程

多线程 # 多线程之间的顺序是不确定的,因为cpu的时间片是极短的

线程与进程的区别:
  进程是分配资源的最小单位,线程是程序执行的最小单位(cpu调度的最小单位)
  一个进程至默认有一个线程,没有进程就没有线程
  线程不能独立执行,必须依附于进程
  进程开销大,线程开销小
  进程可以用于多核,线程不能用于多核
